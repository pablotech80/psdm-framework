# Code of Conduct

## Expected Behavior

PSDM Framework is built for serious engineering work. Discussions should stay professional, specific, and useful.

Expected behavior:

- be respectful when disagreeing;
- explain technical trade-offs with evidence;
- keep criticism focused on code, docs, architecture, tests, or release process;
- avoid sharing secrets, credentials, customer data, private repository output, or production incident details in public threads.

## Unacceptable Behavior

Unacceptable behavior includes harassment, personal attacks, discriminatory language, threats, deliberate disruption, or publishing private information without permission.

## Reporting

Until a dedicated public security or moderation contact is published, report conduct issues privately through the repository owner's preferred private channel.

Security vulnerabilities should follow `SECURITY.md`, not public issue comments.

## Enforcement

Maintainers may edit, hide, lock, or remove comments and issues that expose private data or make collaboration unsafe. Repeated or severe violations may lead to blocked participation.
